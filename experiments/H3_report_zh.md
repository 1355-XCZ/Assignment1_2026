# H3：迭代编码的边际收益递减——为什么更多处理反而更差？

## 动机

QANet 的 Model Encoder 是同一组 7 个编码器块的三次迭代应用（权重共享）：

```
M0 →[7 blocks]→ M1 →[同样的 7 blocks]→ M2 →[同样的 7 blocks]→ M3
```

Pointer 层对这三个中间产物做非对称使用：

```
P(start) = softmax(w1 · [M1; M2])     ← M1 + 第 2 次迭代输出
P(end)   = softmax(w2 · [M1; M3])     ← M1 + 第 3 次迭代输出
```

M1 是两个预测的**共享锚点**——同时出现在 start 和 end 的输入中。直觉上，M1 提供两者共需的"基础定位信号"，M2 和 M3 在此基础上做专业化精化。

接线替换实验（eval 时改变 Pointer 的输入组合，编码器不变，全 dev 集 10,465 样本评估）揭示了两个矛盾：

| 配置 | Start 输入 | End 输入 | F1 | ΔF1 |
|---|---|---|---|---|
| original | M1+M2 | M1+M3 | 70.24 | — |
| sym\_M2 | M1+M2 | M1+M2 | 70.01 | −0.23 |
| sym\_M3 | M1+M3 | M1+M3 | 69.70 | −0.54 |
| no\_M1 | M2+M3 | M2+M3 | 69.67 | −0.58 |
| swap | M1+M3 | M1+M2 | 69.42 | −0.82 |
| only\_M1 | M1+M1 | M1+M1 | 63.97 | −6.28 |

**矛盾 1：共享锚点 M1 几乎不重要。**

移除 M1（no\_M1: −0.58 F1）几乎没有影响，但 M1 独挑大梁（only\_M1: −6.28 F1）则是灾难。M1 同时参与两个预测却最不重要，M2/M3 各参与一个却不可或缺——**参与度与重要性反向**。

**矛盾 2：更多处理反而更差。**

M2 单独做两件事（sym\_M2: −0.23）优于 M3 单独做两件事（sym\_M3: −0.54）。M3 经过了 21 层编码器处理（3 × 7 blocks），M2 只经过了 14 层（2 × 7 blocks）。如果迭代处理单调改善表示质量，M3 应 ≥ M2。**但实际结果相反——更多处理的产物在 Pointer 中更差。**

此外，非对称设计的优势极其微弱：original（70.24）仅比 sym\_M2（70.01）好 0.23 F1。把 M3 分配给 end 预测的所谓"专业化"带来的边际收益近乎可忽略。

## 研究问题

为什么更多的迭代处理（M3，21 层）反而不如更少的处理（M2，14 层）？第三次编码器迭代对 start/end 边界信息造成了什么影响？

## 实验设计

三组实验逐步回答上述问题：

**实验 1（确立事实并解释 M1 冗余）** 接线替换已给出行为事实。我们进一步从两个独立角度验证"M1 冗余"的解释：（a）检查 Pointer 权重 w1/w2 中 M1 对应象限的范数——模型是否在训练中学会了降低对 M1 的依赖；（b）计算 M1-M2、M2-M3 的 CKA，测量信息重叠度——如果 M2 已包含 M1 的全部信息，则 M1 在拼接中是冗余的。

**实验 2（M2 vs M3 的边界信息差异）** 实验 1 确认了 M1 冗余，并将问题聚焦到 M2 vs M3。那么 M2 比 M3"好"在哪里？我们用线性探针在 M1/M2/M3 的 token 表示上分别预测 is\_start\_token 和 is\_end\_token。如果 M2 对两者的 AUC 都 ≥ M3 → 直接证实"第三次迭代损害了边界信息的线性可读性"。如果 M3 对 end 更好但对 start 更差 → 存在部分功能分化，但净效果为负。

**实验 3（第三次迭代做了什么？）** 实验 2 测量了信息差异的方向和大小。实验 3 追问机制：第三次迭代的 7 个编码器块对表示做了什么变换，使得边界信息的线性可读性下降？我们通过表征结构分析（token 间区分度、局部对比度、有效秩）刻画 M2→M3 的变换性质。如果 M3 比 M2 更"平滑"（相邻 token 更相似、区分度更低），说明第三轮 Self-Attention 过度聚合了局部特征，模糊了边界信号。

**目标**：解释矛盾 1——M1 为什么在 Pointer 中不重要？

| 对 | CKA | 含义 |
|---|---|---|
| M1\_M2 | [TBD] | M2 包含多少 M1 的信息 |

**预期**：由于 M2 = f(M1) 且 f 是确定性变换，M2 中包含了 M1 的全部信息——但经过非线性变换后可能以不同"格式"存储。CKA 测量的是线性可对齐的信息重叠。

如果 CKA(M1, M2) 很高（>0.9）→ M1 和 M2 高度重叠，M1 在 [M1;M2] 拼接中确实冗余。

如果 CKA(M1, M2) 中等（0.7-0.9）→ M1 和 M2 存在表示格式差异，但 Pointer 的线性投影仍能从 M2 中提取 M1 的信息（因为 no\_M1 几乎无损失）。

### 关于"信息包含"的讨论

M2 = f(M1) 意味着 M2 中**存在**M1 的全部信息（信息论意义上），但这些信息是否**线性可读**是另一个问题。如果 f 施加了强非线性变换，M1 的信息可能在 M2 中被"加密"到非线性子空间中。

Pointer 是线性投影（w·[M1;M2]），因此它只能利用线性可读的信息。如果 M1 的信息在 M2 中线性不可读，那么保留 M1 应该有帮助——但实验显示移除 M1 几乎无损失。这说明以下两种情况之一成立：（a）M1 的信息在 M2 中仍然线性可读（f 近似保持线性结构），或（b）M1 的信息对 Pointer 的边界预测本身就不太有用（only\_M1 的惨淡表现部分支持这一点，但 only\_M1 也受分布不匹配影响）。

实验 2 的线性探针可以帮助区分这两种解释。

### 实验 1 小结

[根据实际结果填写] 权重分析和 CKA 从参数侧和信息侧双重验证 M1 冗余假设。M1 的贡献已被后续迭代产物"吸收"，共享锚点的设计虽存在但不必要。这一结论将问题聚焦到真正重要的比较：M2 vs M3。

## 实验 2：M2 vs M3 的边界信息差异

**问题**：实验 1 将问题聚焦到 M2 vs M3。接线替换显示 M2 > M3（sym\_M2 优于 sym\_M3），但这是在 Pointer 权重分布不匹配的条件下测量的。我们需要不依赖 Pointer 权重的直接信息测量。

**方法**：在 M1/M2/M3 的 per-token 表示上训练逻辑回归：
- **is\_start\_token**：该 token 是否为答案 start 位置
- **is\_end\_token**：该 token 是否为答案 end 位置

500 样本，80/20 划分，ROC-AUC。

**为什么线性探针适合这个问题**：Pointer 本身就是线性投影（w · [M; M']），因此线性探针与 Pointer 测试的是**同类型**的信息可读性。如果某信息对线性探针不可读，对 Pointer 大概率也不可读。

**结果与假说的对应**：

| 如果发现… | 说明 |
|---|---|
| M2 对 start 和 end 都 > M3 | 第三次迭代损害了边界信息 → 支持"过度处理" |
| M3 > M2 对两者 | sym\_M2 > sym\_M3 是分布不匹配的假象，M3 信息更丰富 |
| M2 更好 start，M3 更好 end | 存在功能分化，但被 sym 实验的分布不匹配掩盖 |
| M2 ≈ M3 | 差异极小，sym\_M2 > sym\_M3 仅反映训练偏差 |

**与接线替换的交叉验证**：

如果探针发现 M2 > M3（信息层面），且接线替换发现 sym\_M2 > sym\_M3（行为层面）→ 两个独立方法一致，结论稳健。

如果探针发现 M3 ≥ M2，但接线替换仍然 sym\_M2 > sym\_M3 → 差异来自 Pointer 权重的分布适配而非信息差异。原始设计中 w1 在 [M1;M2] 上训练更久（start loss 通常比 end loss 更稳定），可能使 w1+M2 的配合优于 w2+M3。

### 局限性

- is\_start 和 is\_end 标签极端不平衡（每样本约 1/400 正样本）
- 线性探针在单独的 M2 或 M3 上操作，但 Pointer 处理 [M1;M2] 的拼接——联合信息可能大于各自之和
- 探针检测的是表示中的信息**存在性**，不是 Pointer 实际**使用**了多少

## 实验 3：第三次迭代对表示做了什么？

**问题**：如果实验 2 确认 M2 > M3（边界信息的线性可读性下降），那么第三次迭代的 7 个编码器块做了什么变换导致了这一下降？

每个编码器块的流水线是 Conv\_0 → Conv\_1 → Self-Attention → FFN。Self-Attention 是唯一的全局操作（其他都是局部的）。如果第三轮迭代的 Self-Attention 将 token 表示向全局均值方向聚合（"过度平滑"），相邻 token 的表示会变得更相似，导致精确的边界位置（"答案从这个 token 开始"）变得更难用线性分类器区分。

**方法**：对 M1/M2/M3 计算以下表征结构指标（200 样本）：

1. **局部对比度**：1 − cosine\_sim(token\_t, token\_{t+1}) 的平均值。高 = 相邻 token 更可区分。
2. **答案边界锐度**：答案边界位置（y1, y2）与相邻位置的余弦距离。高 = 边界更"锐利"。
3. **token 范数变异系数（CoV）**：各 token L2 范数的标准差/均值。高 = 表示中有更多位置选择性激活。

|---|---|---|---|---|

| 答案边界锐度 | [TBD] | [TBD] | [TBD] | [TBD] |
| 范数 CoV | [TBD] | [TBD] | [TBD] | [TBD] |

**预期推理**：

如果 M3 的局部对比度 < M2 → 第三轮迭代确实平滑了相邻 token 的差异（"过度聚合"）

如果 M3 的答案边界锐度 < M2 → 平滑效应特别影响了答案边界位置

如果 M3 的范数 CoV < M2 → 表示变得更均匀、位置选择性降低

这三个指标从不同角度测量"第三次迭代是否模糊了局部特征"。如果三者一致指向 M3 更平滑 → 机制是清楚的：Self-Attention 的全局聚合在第三轮过度了。

### 与 H1 发现的连接

H1 发现 Conv\_1 是 Model Encoder 中最重要的子层——它产生高区分度的局部特征，供 Self-Attention 使用。如果第三轮迭代中 Self-Attention 过度平滑了 Conv\_1 辛苦建立的局部对比度，则 H3 的发现与 H1 形成呼应：Conv\_1 建立局部特征 → Self-Attention 利用并可能过度消耗这些特征 → 迭代次数过多导致边际收益递减直至为负。

### 局限性

- 表征结构分析是描述性的（相关性），不是因果性的。M3 更平滑不等于"因为更平滑所以更差"——可能有混淆变量。
- 编码器块中的 Conv 和 FFN 也对表示有影响，把变化归因于 Self-Attention 的"过度聚合"需要更细粒度的分析（逐子层测量），计算开销较大。
- M1、M2、M3 使用相同的编码器参数，所以每次迭代施加的变换是相同的函数——但作用在不同的输入分布上。"过度平滑"不是 Self-Attention 本身的问题，而是同一变换迭代应用导致的累积效应。

## 综合

### 回答研究问题

**研究问题**：为什么更多的迭代处理（M3）反而不如更少的处理（M2）？第三次编码器迭代对 start/end 边界信息造成了什么影响？

**回答**：[根据实际结果，从以下方向构建]

**如果"过度平滑"假设成立**（M2 探针 AUC > M3，且 M3 局部对比度 < M2）：

M1 在 Pointer 中不重要，因为其信息已被 M2 包含（CKA 验证）。M2 比 M3 更适合边界预测，因为第三次迭代的编码器（特别是 Self-Attention 的全局聚合）平滑了 token 间的局部差异，降低了边界位置的锐度。这意味着 QANet 的三次迭代设计存在**边际收益递减**：第二次迭代（M1→M2）提供了大部分增量信息（only\_M1 vs sym\_M2：6.05 F1 差距），第三次迭代（M2→M3）的边际贡献近乎为零甚至为负。

非对称接线（M2→start, M3→end）的微弱优势（0.23 F1）可能不是因为 M3 对 end 有"专业化"，而是因为第三次迭代在 M2 的基础上做了微小的全局信息整合，这对 end 预测（end 边界通常需要更多上下文）略有帮助，但代价是损失了部分局部锐度。净效果接近零。

**如果"过度平滑"假设不成立**（M3 探针 AUC ≥ M2，或表征结构无差异）：

sym\_M2 > sym\_M3 的差异来自 Pointer 权重的训练偏差，而非 M2/M3 的内在信息差异。模型在训练中对 w1（start 预测）进行了更充分的优化（可能因为 start 边界更锐利、loss 下降更快），使得 w1+M2 的组合在 eval 时比 w2+M3 更稳健。在这种解释下，M2 > M3 是**训练动态**的产物而非**表示质量**的反映。

### 证据链

1. **行为层面**（接线替换）：M1 冗余（−0.58 F1），M2 > M3 做通用表示（−0.23 vs −0.54），非对称优势微弱（0.23 F1）。
2. **参数与信息层面**（实验 1）：Pointer 权重象限分析 + CKA 验证 M1 冗余假设——后续迭代包含了前序迭代的信息。
3. **信息层面**（实验 2）：线性探针直接测量 M2 vs M3 的 start/end 信息，绕过 Pointer 权重，区分"信息差异"和"适配差异"。
4. **机制层面**（实验 3）：表征结构分析刻画 M2→M3 的变换性质——是否存在过度平滑。

### 局限性

1. **eval 时接线替换的分布不匹配**：所有接线替换在 eval 时进行。Pointer 权重在特定输入分布上训练，替换后的输入分布不同。sym\_M2（w2 接收 [M1;M2] 而非 [M1;M3]）和 sym\_M3（w1 接收 [M1;M3] 而非 [M1;M2]）都有 50% 的输入分布不匹配。两者的差异（0.31 F1）中，有多少来自 M2/M3 的内在差异、多少来自分布不匹配的不对称效应，无法完全分离。实验 2 的线性探针部分缓解了这一问题。

2. **因果归因的限制**：实验 3 的表征分析是描述性的。"M3 更平滑"和"M3 边界信息更差"之间的关系是相关性，不是因果性。完全的因果验证需要在第三轮中选择性地关闭 Self-Attention 并测量边界锐度变化——这需要对编码器的逐子层干预，实现复杂度较高。

3. **Stochastic Depth 的训练时效应**：训练时 stochastic depth 按 dropout × l/L 的概率跳过子层（深层跳过概率更高）。M3 经过的第 15-21 层在训练中被跳过的概率最高。这可能导致 M3 的表示在训练中不如 M2 充分优化——"更差"可能部分来自训练不充分而非过度平滑。这一假说超出当前实验的测试范围，但应作为替代解释保留。

### 方法学说明

接线替换实验的一个结构性发现是：**M1 在 Pointer 中是冗余的**。这与 H1 的发现形成对比——H1 证明了某些组件（Conv\_1）不可或缺，而 H3 证明了某个组件（M1 在 Pointer 中的角色）可以安全移除。两者共同表明：QANet 的设计中存在冗余（M1）和关键瓶颈（Conv\_1）并存的现象，理解这种不均匀性是分析深度网络的核心课题之一。
