# H3：Pointer 非对称接线的必要性——迭代编码中的功能分化检验

## 动机

QANet 的 Model Encoder 将同一组 7 个编码器块迭代应用三次（权重共享），产生三个中间表征：

```
M0 →[7 blocks]→ M1 →[同样的 7 blocks]→ M2 →[同样的 7 blocks]→ M3
```

Pointer 层对这三个产物做**非对称使用**：

```
P(start) = softmax(w1 · [M1; M2])     ← 第 1 + 第 2 次迭代
P(end)   = softmax(w2 · [M1; M3])     ← 第 1 + 第 3 次迭代
```

**设计溯源**：QANet 论文对此设计仅表示"We adopt the strategy of Seo et al. (2016)"，未提供任何理论论证。溯源到 BiDAF（Seo et al., 2016），非对称接线的原始形式是：start 预测使用建模层输出 M，end 预测将 M 再经过一层**独立参数**的 BiLSTM 得到 M2 后使用。在 BiDAF 中，这一非对称至少在结构上有合理性——额外的独立参数 BiLSTM 可以学到 end 专用的新特征。但 QANet 将这一模式改编为**共享权重**的编码器迭代：第三次迭代施加的是与前两次完全相同的变换函数，"额外处理产生专业化信息"的前提因此显著弱化。这一从未被正式检验的设计选择，隐含了三个假设：

1. **深度分化**：M2（14 层）和 M3（21 层）携带不同信息，分别适合不同的预测任务——M2 的中等深度适合定位 start，M3 的额外处理为 end 提供专业化精化
2. **共享锚点**：M1（7 层）提供两个预测共需的基础定位信号
3. **迭代增益**：更多的编码器迭代产生更好或至少不同的表示

这些假设是否成立？特别是在共享权重的条件下——同一变换函数的迭代应用能否产生功能分化？如果非对称接线的三个组成部分中有任何一个不成立，该设计选择就缺乏充分的理由。

## 研究问题

Pointer 非对称接线所隐含的功能分化假设——M1 作为共享锚点提供基础信号、M2 与 M3 因处理深度不同而各有专长——是否在训练后的模型中成立？

## 方法学说明：评估时干预与机制分析

本研究的所有实验均在**评估时**进行——接线替换、消融和信息探测都在完整训练好的模型上执行，不重新训练。这是一个有意识的方法论选择：

我们的研究目标是**机制分析**——理解已训练模型中各组件当前承担的功能角色。这与 Meng et al.（2022）在 GPT 中定位事实关联所采用的因果追踪方法论一致：在训练好的模型上施加干预，观察模型行为的变化，定位特定计算步骤的功能角色。训练时消融会让模型在训练过程中学会补偿，抹去被移除组件原本承担的功能痕迹，反而不适合机制分析。

接线替换是 eval 时干预的一种形式：编码器不变，仅改变 Pointer 的输入组合。这直接测量的是"模型当前如何使用各迭代产物"。由于 Pointer 权重在特定输入分布上训练，替换后存在分布不匹配——这一固有局限在实验 2 中通过不依赖 Pointer 权重的线性探针进行缓解。

## 前置验证：非对称设计的优势有多大？

我们通过接线替换直接检验非对称设计的必要性。在 eval 时改变 Pointer 的输入组合（编码器不变），在全 dev 集 10,465 样本上评估。

| 配置 | Start 输入 | End 输入 | F1 | ΔF1 | 检验的假设 |
|---|---|---|---|---|---|
| original | M1+M2 | M1+M3 | 70.24 | — | 基线（非对称设计） |
| sym\_M2 | M1+M2 | M1+M2 | 70.01 | −0.23 | M3 是否必要？ |
| sym\_M3 | M1+M3 | M1+M3 | 69.70 | −0.54 | M2 是否更好？ |
| no\_M1 | M2+M3 | M2+M3 | 69.67 | −0.58 | M1 是否必要？ |
| swap | M1+M3 | M1+M2 | 69.42 | −0.82 | 分配方向是否重要？ |
| only\_M1 | M1+M1 | M1+M1 | 63.97 | −6.28 | M2/M3 是否必要？ |

三个设计假设的检验结果：

**假设 1（深度分化）——几乎不成立**：非对称设计（original）仅比对称设计（sym\_M2）好 **0.23 F1**。将 M3 分配给 end 预测的所谓"专业化"带来的边际收益近乎可忽略。更值得注意的是，M2 做两件事（sym\_M2: −0.23）**优于** M3 做两件事（sym\_M3: −0.54）——更多处理的产物反而更差。

**假设 2（共享锚点）——不成立**：移除 M1（no\_M1: −0.58）几乎没有影响，但 M1 独挑大梁（only\_M1: −6.28）则是灾难。M1 同时参与两个预测却最不重要——共享锚点是冗余设计。

**假设 3（迭代增益）——部分成立但存在递减**：M2/M3 是必需的（only\_M1: −6.28），但 M2 已捕获绝大部分增量价值。第二次迭代的增量（only\_M1 vs sym\_M2: 6.05 F1）远大于第三次迭代的边际贡献（接近零甚至为负）。

**关于 swap 的说明**：swap（−0.82）是所有非 only\_M1 配置中损失最大的，但这不说明 M2/M3 携带不同信息。swap 让 w1 和 w2 **同时**接收训练时未见过的分布（w1 训练在 [M1;M2]，swap 后收到 [M1;M3]；w2 反之），而 sym\_M2 只有 w2 不匹配（0.23），sym\_M3 只有 w1 不匹配（0.54）。如果两者的不匹配效应独立，预期 swap ≈ 0.23 + 0.54 = 0.77，实际 0.82，吻合度很高。swap 的损失来自**双重分布不匹配**，不是 M2/M3 的信息差异。

**结论**：非对称设计的三个假设中，深度分化和共享锚点的预期收益均未在行为层面体现，迭代增益在第三次迭代已基本耗尽。接下来的实验探究**为什么**这些假设未得到支持。

## 实验设计

三组实验逐步解释前置验证中观察到的三个设计失效：

**实验 1（共享锚点为什么冗余？）** 从参数侧和信息侧验证 M1 冗余的原因：（a）Pointer 权重中 M1 象限的范数——模型是否学会了降低对 M1 的依赖；（b）M1-M2 的 CKA——M2 是否已吸收 M1 的信息。

**实验 2（M2 和 M3 的信息是否不同？）** 前置验证显示 sym\_M2 > sym\_M3，但这受 Pointer 权重分布不匹配的影响。我们用不依赖 Pointer 权重的线性探针直接测量 M1/M2/M3 的 start/end 边界信息，判断 M2 > M3 是内在信息差异还是适配偏差。

**实验 3（第三次迭代做了什么？）** 即使实验 2 显示 M2 和 M3 的边界信息几乎相同，第三次迭代是否仍然改变了表征结构？我们通过表征结构分析刻画 M2→M3 的变换性质。

---

## 实验 1：共享锚点为什么冗余？

**目标**：解释前置验证中的发现——M1 在 Pointer 中几乎不重要。

### 1a. Pointer 权重象限分析

**方法**：Pointer 的 w1 形状为 [2d]（d=96），其中前 d 维对应 M1 的特征，后 d 维对应 M2 的特征（拼接顺序为 [M1; M2]）。同理，w2 的前 d 维对应 M1，后 d 维对应 M3。

| 权重 | 前 d 维（M1 象限） | 后 d 维（M2/M3 象限） | 比值 |
|---|---|---|---|
| w1 | 0.0902 | 0.1773 | 1.97× |
| w2 | 0.0728 | 0.1232 | 1.69× |

w1 和 w2 的 M2/M3 象限范数分别是 M1 象限的 **1.97 倍**和 **1.69 倍**。一个自然的解读是模型在训练中学会了对 M1 赋予更低的权重。**但需注意**：线性模型中权重范数受输入特征幅度影响——如果 M1 的 token 表示平均幅度系统性地大于 M2/M3，权重较小可能仅是尺度补偿而非依赖度的反映。当前实验未报告 M1/M2/M3 的绝对特征幅度，因此权重象限比值应解读为**方向性支持**（与行为证据一致），而非独立的强证据。

### 1b. CKA 信息重叠

**方法**：计算 M1-M2、M2-M3、M1-M3 的线性 CKA（全 dev 集）。

| 对 | CKA | 含义 |
|---|---|---|
| M1\_M2 | 0.871 | M2 包含大部分 M1 的线性可读信息 |
| M2\_M3 | 0.954 | M2 和 M3 几乎共享相同的表征结构 |
| M1\_M3 | 0.717 | M3 与 M1 的距离最远——迭代逐步变换表征 |

CKA(M1, M2) = 0.871 表明 M1 和 M2 之间存在一定的线性结构差异（非 >0.9 的近乎相同），M2 并没有完美保留 M1 的线性格式。然而，no\_M1 仅造成 −0.58 F1 损失——这说明 M1 中那些"未被 M2 线性保留"的信息（约 13% 的 CKA 缺口）对 Pointer 的边界预测并不重要。Pointer 需要的所有有用信息已经在 M2 中以线性可读的形式存在。

CKA(M2, M3) = 0.954 表明第三次迭代几乎没有改变表征的线性结构——M2 和 M3 本质上携带相同的信息。

### 实验 1 小结

权重分析和 CKA 从两个独立角度解释 M1 冗余：

- **参数侧**：Pointer 权重的 M2/M3 象限范数是 M1 象限的 1.7-2.0 倍，方向上与 M1 低重要性一致，但权重范数可能受输入特征尺度影响，不作为独立强证据。
- **信息侧**：CKA(M1, M2) = 0.871 说明 M2 与 M1 共享大量线性结构。结合 no\_M1 仅 −0.58 F1 的行为证据，M1 在 [M1;M2] 拼接中提供的增量价值可忽略。

共享锚点设计虽存在但提供的增量价值可忽略——这是非对称设计的**第一个未得到验证的假设**。

## 实验 2：M2 和 M3 的信息是否不同？

**目标**：直接测量 M2 vs M3 的边界信息差异，不依赖 Pointer 权重。这检验非对称设计的核心假设——不同深度的迭代产物是否携带不同信息。

**方法**：在 M1/M2/M3 的 per-token 表示上训练逻辑回归：
- **is\_start\_token**：该 token 是否为答案 start 位置
- **is\_end\_token**：该 token 是否为答案 end 位置

500 样本，80/20 划分，ROC-AUC。

**为什么线性探针适合这个问题**：Pointer 本身就是线性投影（w · [M; M']），因此线性探针与 Pointer 测试的是**同类型**的信息可读性。如果某信息对线性探针不可读，对 Pointer 大概率也不可读。

| 表示 | is\_start AUC | is\_end AUC |
|---|---|---|
| M1 | 0.977 | 0.979 |
| M2 | **0.979** | **0.981** |
| M3 | 0.978 | 0.980 |

**核心发现：未检测到功能分化。** M2 对 start **和** end 都取得了最高 AUC，M3 对两者都略低于 M2。预期的"M2 擅长 start、M3 擅长 end"的专业化模式**未出现**。

所有差异均在 ΔAUC = 0.001 之内。作为参考，线性探针在 ~14,000 测试 token（其中仅 ~100 正样本）上评估，这个样本量下 AUC 的典型标准误约为 0.005-0.01——0.001 的差异远小于一个标准误，属于统计噪声范围。三个迭代产物的边界信息含量在统计意义上不可区分。渐进模式为 M1→M2 提升（+0.002）、M2→M3 微降（−0.001）——第二次迭代略微改善了线性可读的边界信息，第三次迭代则小幅回退。

**与接线替换的交叉验证**：探针（信息层面）和接线替换（行为层面）一致指向 M2 ≥ M3。两个方法的收敛增强了结论的稳健性：在当前实验的检测范围内，M2 和 M3 之间**未观察到有意义的功能分化**，非对称接线缺乏信息层面的支撑——这是非对称设计的**第二个未得到验证的假设**。

### 局限性

- is\_start 和 is\_end 标签极端不平衡（约 0.70% 正样本）
- 线性探针在单独的 M2 或 M3 上操作，但 Pointer 处理 [M1;M2] 的拼接——联合信息可能大于各自之和
- 探针检测的是表示中的信息**存在性**，不是 Pointer 实际**使用**了多少

## 实验 3：第三次迭代对表示做了什么？

**目标**：实验 2 显示 M2 和 M3 的边界信息几乎相同（ΔAUC = 0.001），但 CKA(M2, M3) = 0.954 而非 1.0——第三次迭代确实改变了某些东西。它改变了什么？

每个编码器块的流水线是 Conv\_0 → Conv\_1 → Self-Attention → FFN。Self-Attention 是唯一的全局操作。如果第三轮迭代的 Self-Attention 将 token 表示向全局均值方向聚合（"过度平滑"），相邻 token 的表示会变得更相似，导致精确的边界位置变得更难用线性分类器区分。

**方法**：对 M1/M2/M3 计算以下表征结构指标（200 样本）：

1. **局部对比度**：1 − cosine\_sim(token\_t, token\_{t+1}) 的平均值。高 = 相邻 token 更可区分。
2. **答案边界锐度**：答案边界位置（y1, y2）与相邻位置的余弦距离。高 = 边界更"锐利"。
3. **token 范数变异系数（CoV）**：各 token L2 范数的标准差/均值。高 = 表示中有更多位置选择性激活。

| 指标 | M1 | M2 | M3 | M2→M3 变化 |
|---|---|---|---|---|
| 局部对比度 | 0.1162 | 0.0669 | 0.0480 | −28.2% |
| 答案边界锐度 | 0.2213 | 0.1354 | 0.1017 | −24.9% |
| 范数 CoV | 0.1627 | 0.1361 | 0.1273 | −6.5% |

**三个指标一致指向 M3 更平滑**：局部对比度下降 28%，答案边界锐度下降 25%，范数多样性下降 6.5%。渐进模式 M1→M2→M3 呈现持续的平滑趋势——每次迭代都在降低局部区分度。

**为什么 M1→M2 平滑更多，信息反而提升？** 完整的渐进模式值得审视：

| 迭代 | 局部对比度变化 | 探针 AUC 变化（start） |
|---|---|---|
| M1→M2 | 0.116 → 0.067（**−42%**） | 0.977 → 0.979（**+0.002**） |
| M2→M3 | 0.067 → 0.048（−28%） | 0.979 → 0.978（−0.001） |

第二次迭代的平滑幅度比第三次**更大**（42% vs 28%），但它同时提升了边界信息。这看似矛盾，实则不然：M1 是 CQ Attention 输出经过第一轮编码的结果，表征中仍保留大量局部噪声。第二次迭代的平滑是**有益的信息压缩**——在过滤噪声的同时整合了跨位置的语义关系，使边界信息更加线性可读。而到了 M2，表征已经接近该共享变换函数的**近似不动点**（CKA(M2, M3) = 0.954），有用的信息压缩已经完成。第三次迭代继续平滑但**没有新信息可整合**——是"无回报的平滑"。

**替代假设：stochastic depth 导致的训练不充分。** 上述"无回报的平滑"解释假定第三次迭代的变换在功能上是冗余的。但存在一个竞争性解释：训练时 stochastic depth 按 dropout × l/L 的概率跳过子层（深层跳过概率更高）。M3 经过的第 15-21 层在训练中被跳过的概率最高，这可能导致 M3 的表征不是被"过度平滑"，而是**训练不充分**——第三次迭代的参数没有获得足够的梯度更新来学习有用的变换。如果这一解释成立，M2→M3 的结构退化就不是"共享权重迭代的固有缺陷"，而是"正则化的副作用"。当前实验无法区分这两个假设，因为两者都预测 M3 表征质量低于 M2。区分它们需要在关闭 stochastic depth 的条件下重新训练并比较——这超出当前实验范围。我们将两者并列保留。

**关键张力：结构大幅改变，但信息几乎不变。** 第三次迭代的表征结构变化显著（20-28%），但线性探针 AUC 仅变化 0.001。这意味着：第三次迭代确实大幅平滑了表征的几何结构——相邻 token 更相似、边界更模糊、范数更均匀——但边界位置的信息以更分散的编码方式**保留了下来**，线性分类器仍能提取它。信息在，但"格式"变了。

这解释了 sym\_M2 和 sym\_M3 之间仅 0.31 F1 的差异：M3 的表征虽然在结构上比 M2 更平滑，但 Pointer 的线性投影仍能从中提取足够的边界信号——只是效率略低。

### 与 H1 发现的连接

H1 发现 Conv\_1 是 Model Encoder 中最重要的子层——它产生高区分度的局部特征，供 Self-Attention 使用。第三轮迭代中 Self-Attention 的全局聚合确实平滑了 Conv\_1 建立的局部对比度（−28%），与 H1 形成呼应：Conv\_1 建立局部特征 → Self-Attention 利用并逐步消耗这些特征 → 迭代次数过多导致表征结构退化，尽管信息并未丢失。

### 局限性

- 表征结构分析是描述性的（相关性），不是因果性的。M3 更平滑不等于"因为更平滑所以更差"——可能有混淆变量。
- 编码器块中的 Conv 和 FFN 也对表示有影响，把变化归因于 Self-Attention 的"过度聚合"需要更细粒度的逐子层分析。
- M1、M2、M3 使用相同的编码器参数，每次迭代施加的是相同的函数作用在不同的输入分布上。"过度平滑"不是 Self-Attention 本身的问题，而是同一变换迭代应用导致的累积效应。

## 综合

### 回答研究问题

**研究问题**：Pointer 非对称接线所隐含的功能分化假设——M1 共享锚点、M2/M3 深度专业化——是否在训练后的模型中成立？

**回答：在训练后的模型中未得到支持。** 非对称设计的三个假设均未在当前实验中获得证实：

**1. 共享锚点——近乎冗余。** 替换 M1 仅造成 −0.58 F1（行为层面）。Pointer 权重对 M1 象限赋予了较低的范数（参数层面，M2/M3 象限是 M1 的 w1: 1.97×, w2: 1.69×）。CKA(M1, M2) = 0.871 表明 M2 与 M1 共享大量线性结构（信息层面）。三个角度一致指向同一方向：**M1 在 [M1;M2] 拼接中提供的增量价值可忽略**。需注意 no\_M1 实验是用 M2/M3 替代 M1 位置（非纯零化），−0.58 的损失部分反映了替代表征的相似性。

**2. 深度专业化——未检测到。** 线性探针显示 M2 对 start（AUC=0.979）和 end（AUC=0.981）都取得最高分，M3 对两者都略低（start=0.978, end=0.980）。预期的"M2→start, M3→end"专业化模式**未出现**——所有差异在 ΔAUC=0.001 之内，远小于估计标准误（~0.005-0.01），属统计噪声范围。接线替换（行为层面）与探针（信息层面）一致指向 M2 ≥ M3。需注意：探针测试的是单独的 M2/M3，不是 Pointer 实际使用的 [M1;M2] 拼接；且线性探针可能无法捕捉非线性编码的专业化。

**3. 迭代增益——第二次迭代捕获绝大部分价值，第三次迭代的边际贡献接近零。** only\_M1 vs sym\_M2 的 6.05 F1 差距说明第二次迭代是必需的；但第三次迭代的边际 F1 贡献几乎为零。实验 3 观察到一个值得注意的现象：第三次迭代进一步平滑了表征结构（局部对比度 −28%，边界锐度 −25%），但线性可读的边界信息几乎不变（ΔAUC=0.001）。信息以更分散的编码方式保留，**结构变化不等于信息丢失**。不过，M3 的结构变化可能部分源自 stochastic depth 导致的训练不充分（见局限性讨论），而非迭代本身的固有效应。

非对称接线（M2→start, M3→end）仅比对称接线（sym\_M2）好 0.23 F1。这一微弱优势在线性探针所测的边界信息中找不到支撑（ΔAUC=0.001），更可能来自 Pointer 权重对训练分布的适配。

### 证据链

1. **行为层面**（前置验证）：非对称优势仅 0.23 F1；M1 冗余（−0.58）；M2 > M3 通用性（sym\_M2 > sym\_M3）；swap 的 −0.82 可由双重分布不匹配解释（0.23 + 0.54 ≈ 0.77 vs 实际 0.82）。
2. **参数层面**（实验 1a）：w1/w2 的 M2/M3 象限范数是 M1 象限的 1.7-2.0 倍，方向上与 M1 低重要性一致（但权重范数可能受输入特征尺度影响，为方向性而非独立证据）。
3. **信息层面**（实验 1b + 实验 2）：CKA 确认 M1 信息被 M2 吸收（0.871），M2 和 M3 高度相似（0.954）。线性探针确认 M2/M3 无功能分化（ΔAUC=0.001）。
4. **机制层面**（实验 3）：第三次迭代大幅平滑表征结构（20-28%），但不破坏线性可读的边界信息——信息保留而格式改变。

### 与其他假说的连接

H1 证明了 Conv\_1 是 Model Encoder 中不可或缺的组件（移除后 −32.54 F1），H3 证明了 M1 在 Pointer 中是冗余的（移除后仅 −0.58 F1）。两者共同表明 QANet 的设计中存在**关键瓶颈与冗余并存**的现象。H2 证明了 CQ Attention 中 C2Q 方向不可替代而 Q2C 冗余——同样是"不可替代性 vs 冗余"的主题。三个假说从不同层面揭示了 QANet 内部功能分布的不均匀性。

实验 3 与 H1 的直接呼应尤为显著：H1 发现 Conv\_1 建立局部特征是 Model Encoder 的核心功能，H3 发现迭代应用的 Self-Attention 逐步消耗这些局部特征（M1→M2→M3 局部对比度从 0.116 降至 0.048）。这构成了一个完整的图景：Conv\_1 生产、Self-Attention 消费、迭代过多导致消费过度。

### 局限性

1. **eval 时接线替换的分布不匹配**：Pointer 权重在特定输入分布上训练，接线替换后输入分布不同。sym\_M2 和 sym\_M3 都有 50% 的输入分布不匹配。两者的差异（0.31 F1）中，有多少来自 M2/M3 的内在差异、多少来自分布不匹配的不对称效应，无法完全分离。实验 2 的线性探针部分缓解了这一问题。

2. **因果归因的限制**：实验 3 的表征分析是描述性的。"M3 更平滑"和"边界信息格式改变"之间的关系是观察到的共现，不是因果关系。完全的因果验证需要在第三轮中选择性地关闭 Self-Attention 并测量结构变化，实现复杂度较高。

3. **Stochastic Depth 的训练时效应**：如实验 3 正文所述，M3 的结构变化可能部分源自 stochastic depth 导致的训练不充分而非迭代过度平滑。当前实验无法区分两者，这是结论 3（迭代增益耗尽）最主要的替代解释。
