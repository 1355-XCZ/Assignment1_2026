# H2：CQ Attention 的方向不对称性——从 BiDAF 到 QANet 的跨架构验证

## 动机

QANet 从 BiDAF（Seo et al., 2017）继承了双向 CQ Attention 设计。该层将 Context 和 Question 两条独立编码的信息流融合为一个表征，输出是四个子组件的拼接：

```
输出 = [C;  A;  C⊙A;  C⊙B]     ——共 4d 维
        ↑    ↑    ↑      ↑
       原始  C→Q  门控   Q→C→C
```

其中 A = S1·Q（C→Q 方向），C⊙B 来自 B = S1·S2ᵀ·C（Q→C 方向），S1 沿 Q 维度 softmax，S2 沿 C 维度 softmax。两个方向共享同一个相似度矩阵 S 和可训练权重 w，区别仅在于 softmax 的归一化方向。

BiDAF（Seo et al., 2017, Table 1b）通过训练时消融（移除组件后重训练模型）测量了两个方向的相对重要性：

| 条件 | F1 | ΔF1 |
|---|---|---|
| BiDAF (single) | 77.3 | — |
| 移除 C2Q (A) | 67.7 | −9.6 |
| 移除 Q2C (B) | 73.7 | −3.6 |

C2Q 方向的重要性约为 Q2C 的 **2.7 倍**。然而 BiDAF 使用 LSTM 作为编码器，QANet 用卷积和自注意力完全替换了 LSTM。**架构改变了，不对称性是否仍然成立？**

这一问题并非显然：QANet 的 Model Encoder（7 blocks × 3 passes = 21 层）比 BiDAF 的双层 LSTM 有远强的后续处理能力，可能能够从较弱方向的粗略信号中补偿信息。如果 QANet 的强编码器"抹平"了方向差异，则不对称性是 LSTM 容量瓶颈的产物；如果不对称性持续甚至加剧，则它是注意力融合机制的固有属性。

此外，BiDAF 的消融粒度为方向级（C2Q vs Q2C），未分离 [C, A, C⊙A, C⊙B] 四个子组件的独立贡献。我们有机会以更细的粒度重新审视这一问题。

## 研究问题

BiDAF 发现的 C2Q > Q2C 方向不对称性（2.7×）在 QANet 的 Conv+Attention 架构下是否成立？如果是，什么机制在不同架构下维持了它？

## 方法学说明：评估时干预与机制分析

BiDAF 的消融实验（Seo et al., 2017, Table 1b）采用**训练时消融**——移除组件后重新训练整个模型。这回答的是架构设计问题：*该组件在架构上是否可替代？*

我们选择 **eval 时消融**——在完整训练好的模型上直接零化子组件输出。这是一个有意识的方法论选择，而非资源限制下的退而求其次。原因如下：

我们的研究目标是**机制分析**——理解已训练模型中各子组件当前承担的功能角色——而非架构搜索。这与 Meng et al.（2022）在 GPT 中定位事实关联所采用的因果追踪方法论一致：在训练好的模型上施加干预（腐蚀 + 恢复），观察模型行为的变化，从而定位特定计算步骤的功能角色。训练时消融会让模型在训练过程中学会补偿，抹去被移除组件原本承担的功能痕迹，反而**不适合**机制分析。

因此，eval 时消融测量的是"模型当前如何使用该组件"——这正是我们需要回答的问题。

**与 BiDAF 的比较如何成立**：由于方法不同，两者的 ΔF1 **绝对值不可直接比较**（eval 时消融因分布偏移倾向于产生更大的性能下降）。但方法论偏差对 C2Q 和 Q2C 两个方向的影响近似等比例——两者都是注意力子组件，零化造成的分布偏移程度相当。因此 **C2Q/Q2C 方向比值**是跨方法稳健的比较指标。我们的比较严格限定在比值层面。

## 前置验证：CQ Attention 是关键瓶颈

在解剖 CQ Attention 内部之前，我们首先验证它作为整体的重要性。CQ Attention 是两条流的唯一融合点——在架构上处于瓶颈位置。我们通过 eval 时零化其完整输出（X = 0）并测量 F1/EM 下降来确认这一点。

| 条件 | F1 | EM | ΔF1 |
|---|---|---|---|
| 基线 | 70.24 | 59.22 | — |
| CQ 输出置零 | 7.11 | — | −63.13 (−89.9%) |

F1 下降 89.9%，远超 50% 阈值。CQ Attention 承载了模型性能所依赖的几乎全部信息，解剖其内部子组件有充分意义。

## 实验 1：方向不对称性是否成立？

**目标**：在子组件级别测量四个输出的因果重要性，汇总到方向级与 BiDAF 交叉比较。

### 1a. 子组件消融

**方法**：CQ Attention 的输出 X 形状为 [B, 4d, L]。四个子组件按维度排列：X[:, :d] 为 C，X[:, d:2d] 为 A，X[:, 2d:3d] 为 C⊙A，X[:, 3d:4d] 为 C⊙B。在 X 送入 cq\_resizer 之前，将目标区间置零，其余不变，在全 dev 集上评估 F1/EM。

| 条件 | 置零内容 | 方向归属 | F1 | ΔF1 |
|---|---|---|---|---|
| 基线 | 无 | — | 70.24 | — |
| −C | 原始 context | 无注意力 | 61.73 | −8.51 |
| −A | S1·Q | C→Q | 66.20 | −4.04 |
| −(C⊙A) | 门控 | C→Q 门控 | 54.41 | −15.83 |
| −(C⊙B) | S1·S2ᵀ·C | Q→C→C | 69.21 | −1.03 |
| 仅保留 C | [C; 0; 0; 0] | 无注意力 | 8.35 | −61.90 |

### 1b. 方向级汇总与 BiDAF 对比

将子组件结果汇总到方向级：C2Q = 同时零化 A 和 C⊙A；Q2C = 零化 C⊙B。

| 比较 | BiDAF (LSTM) | QANet (Conv+Attn) |
|---|---|---|
| −C2Q (移除 A + C⊙A) | −9.6 F1 | −62.78 F1 |
| −Q2C (移除 C⊙B) | −3.6 F1 | −1.03 F1 |
| C2Q / Q2C 比值 | 2.7× | **61.0×** |

### 1c. cq\_resizer 权重分析

**方法**：cq\_resizer 是 Conv1d(4d, d, kernel\_size=1)，权重 W 形状为 [d, 4d]。将 W 按列分为四个 [d, d] 象限，计算各象限的 Frobenius 范数。

消融测量的是移除某组件后的**性能损失**（输出侧）；权重范数测量的是模型对该组件的**学习依赖度**（参数侧）。两者是独立的验证视角：如果消融排序与权重范数排序一致，则结论的可信度显著增强。

| 象限 | 对应子组件 | Frobenius 范数 | % of total |
|---|---|---|---|
| W[:, :d] | C (原始) | 12.42 | 44.6% |
| W[:, d:2d] | A (C→Q) | **18.44** | **66.2%** |
| W[:, 2d:3d] | C⊙A (门控) | 11.90 | 42.8% |
| W[:, 3d:4d] | C⊙B (Q→C) | 11.78 | 42.3% |

### 实验 1 小结

**发现 1（子组件排序）**：四个子组件的因果重要性排序为 **C⊙A (−15.83) ≫ C (−8.51) > A (−4.04) ≫ C⊙B (−1.03)**。门控项 C⊙A 是最重要的单一子组件，Q2C 方向（C⊙B）几乎可忽略。

**发现 2（跨架构比较）**：QANet 的 C2Q/Q2C 比值为 **61.0×**（BiDAF 为 2.7×）。方向不对称性在架构变化后不仅**持续**，而且**剧烈加剧**——从 2.7 倍放大到 61 倍。移除 C2Q 后 F1 跌至 7.46，接近整层移除（7.11），说明 C2Q 几乎承载了 CQ Attention 的全部价值。Q2C（C⊙B）仅贡献 −1.03 F1，在 QANet 中近乎冗余。

**发现 3（权重验证）**：A（C→Q 主项）的 Frobenius 范数（18.44）显著高于其他三个象限（11.78–12.42），从参数侧**支持** C2Q 方向的主导地位。但 C⊙A 与 C⊙B 的范数接近（11.90 vs 11.78），未能预测两者 15 倍的消融差异。权重范数在方向级（A 最大 → C2Q 主导）与消融一致，但在子组件级粒度不足以完全区分功能重要性——子组件接收的输入信号质量同样决定其因果效应。

**发现 4（C 组件的角色）**：原始 Context 直通（C）是第二重要的子组件（−8.51 F1），说明模型同时依赖注意力处理后的信号和未经注意力变换的原始上下文。C 在架构上是方向中性的（不涉及 C2Q 或 Q2C 注意力），不影响方向不对称性的比较。

**干预大小控制**：方向级比较中，C2Q 消融零化了 2/4 维度（A + C⊙A），Q2C 仅零化 1/4 维度（C⊙B），干预规模不同。为排除这一混淆，我们检查单子组件对比（各零化相同的 1/4 维度）：C⊙A (−15.83) vs C⊙B (−1.03) = **15.4×**，A (−4.04) vs C⊙B (−1.03) = **3.9×**。即使在严格控制干预大小后，C2Q 方向的每个子组件均显著优于 Q2C。不对称性在任何粒度下都成立。

## 实验 2：什么机制维持了不对称性？

**前提**：实验 1 确认方向不对称性成立（如果不成立，本实验的问题变为"什么导致了不对称性的消失"，但实验设计不变）。

**问题**：C2Q 方向更重要。但 C2Q 和 Q2C 共享同一个相似度矩阵 S，区别仅在 softmax 方向。为什么仅仅改变归一化方向就产生了如此大的因果差异？

从数学定义出发，两个方向做的事情本质不同：

- **C2Q (A = S1·Q)**：将 Question embeddings **注入** Context 的每个位置——引入 Context 中原本**不存在**的新信息。
- **Q2C (C⊙B, 其中 B = S1·S2ᵀ·C)**：通过 Question 中介，在 Context 位置之间**重新分配** Context 自身的信息——不引入新信息，仅重新组织已有信息。

如果 Context 中与答案预测相关的信息在空间上高度集中（答案区间仅占全部 token 的一小部分），那么"重新分配已有 Context 信息"的价值很低——有用信息已经在正确的位置。而"注入 Question 信息"则不可替代——Question 信号没有其他路径进入 CQ Attention 的输出。

我们通过选择性腐蚀检验信息集中度这一前提。

### 方法

只对 Context 的特定位置施加噪声（σ = 3 × std），其余位置保持干净。200 样本，每样本 3 次噪声重复。

条件：
- **ans\_only**：只腐蚀答案区间位置（y1 到 y2）
- **non\_ans\_only**：只腐蚀非答案、非填充位置
- **full\_context**：腐蚀所有 Context 位置（基线参照）

| 条件 | Mean TE | ±95% CI | Avg #tokens | TE/token |
|---|---|---|---|---|
| ans\_only | 0.170 | ±0.034 | 3.1 | 55.7 × 10⁻³ |
| non\_ans\_only | 0.266 | ±0.041 | 140.5 | 1.89 × 10⁻³ |
| full\_context | 0.361 | ±0.045 | 143.6 | 2.52 × 10⁻³ |

**信息密度比**：TE/token(答案) ÷ TE/token(非答案) = **29.5×**

补充观测：TE(ans) + TE(non) = 0.436 > TE(full) = 0.361，交互项 = −0.075（次可加性），说明答案区间和非答案区间的腐蚀效应有部分重叠。

### 推理

密度比 = 29.5×，远超 5× 阈值。Context 的因果信息**高度集中**在答案区间（平均 3.1 个 token 占据 47% 的总效应）。

这一发现验证了机制论证的前提：

1. **Q2C 的重分配功能近乎冗余**：有用的 Context 信息已经集中在正确的位置，通过 Question 中介重新分配 Context 表征几乎不增加新价值。此外，QANet 的 Model Encoder 包含 21 层 Self-Attention，其功能恰恰是在 Context 位置之间重新分配信息——与 Q2C 高度重叠。Q2C 的功能被下游结构性替代。
2. **C2Q 的注入功能不可替代**：Question 信息在 CQ Attention 之后没有任何其他路径进入 Context 的表征流。无论下游编码器多强，它都无法凭空生成 Question 信号——这只能由 C2Q 提供。
3. **信息集中加剧了不对称性**：信息越集中，"重新分配"的边际价值越低（已在正确位置），而"注入新信号到正确位置"的价值不变。29.5× 的集中度意味着 Q2C 的功能空间被极度压缩。

### 局限性

- 腐蚀答案区间 token 直接移除了目标信号，高 TE 是预期的。真正有信息量的观测是非答案 token 的效应和密度比的具体数值——29.5× 远超基线。
- 信息集中度测量的是 Context 输入端的属性。"新信息注入 vs 已有信息重分配"的功能区分来自 CQ Attention 的数学定义（A = S1·Q 引入 Q，B = S1·S2ᵀ·C 重组 C），是演绎性的，而非实验直接测量的。

## 综合

### 回答

**研究问题**：BiDAF 的 C2Q > Q2C 方向不对称性在 QANet 下是否成立？什么机制维持了它？

**回答**：方向不对称性不仅跨架构稳健，而且在 QANet 中**剧烈加剧**——从 BiDAF 的 2.7× 放大到 **61.0×**。

**事实**：在 QANet 的 Conv+Attention 架构下，C2Q 子组件（A, C⊙A）的联合因果效应是 Q2C（C⊙B）的 61.0 倍。移除 C2Q 后 F1 跌至 7.46（接近整层移除的 7.11），C2Q 几乎承载了 CQ Attention 的全部价值。Q2C 仅贡献 −1.03 F1，在 QANet 中近乎冗余。即使在控制干预大小的单子组件对比下（各零化 1/4 维度），C⊙A vs C⊙B = 15.4×，A vs C⊙B = 3.9×，不对称性在任何粒度都成立。cq\_resizer 权重范数在方向级独立验证了 C2Q 的主导地位。

**机制**：不对称性的根源在于两个方向执行的是**本质不同的运算**：

- **C2Q (A = S1·Q)** 将 Question embeddings 注入 Context 表征——这是 **新信息的唯一来源**。Question 信号在 CQ Attention 之后没有任何其他路径进入 Context 的表征流。无论下游编码器多强，它都无法凭空生成 Question 信号。C2Q 不可替代。
- **Q2C (C⊙B, 其中 B = S1·S2ᵀ·C)** 通过 Question 中介在 Context 位置之间重新分配 Context 信息——不引入新信息，仅重新组织已有信息。这一功能与 QANet Model Encoder 的 21 层 Self-Attention 高度重叠。Q2C 可被替代。

实验 2 验证了这一机制的前提条件：Context 的因果信息高度集中在答案区间（密度比 29.5×，平均 3.1 个答案 token 占据 47% 的总效应）。信息已经在正确的位置——重新分配它几乎没有价值，但注入 Question 信号来"标记"这些位置是不可或缺的。

**放大效应**（2.7× → 61×）：BiDAF 的后续处理仅有 2 层 LSTM，无法充分替代 Q2C 的 Context 重分配功能，因此 Q2C 在 BiDAF 中仍有 −3.6 F1 的残余价值。QANet 拥有 21 层包含 Self-Attention 的 Model Encoder，其 Context 内部重分配能力远超 Q2C，将 Q2C 的残余价值压缩至 −1.03 F1（近乎零）。C2Q 的不可替代性在两种架构下不变，Q2C 的冗余度随编码器增强而递增——因此比值从 2.7× 放大到 61×。

### 证据链

1. **前置验证**：CQ Attention 整体消融导致 F1 下降 **89.9%**（70.24 → 7.11），确认其信息瓶颈角色，解剖内部子组件有意义。
2. **核心事实**（实验 1）：子组件消融确立排序 C⊙A ≫ C > A ≫ C⊙B；方向级汇总确立 C2Q/Q2C = **61.0×**（BiDAF 为 2.7×）；单子组件对比在控制干预大小后仍成立（C⊙A/C⊙B = 15.4×）；cq\_resizer 权重 A 象限范数最高（18.44），从参数侧独立验证。
3. **机制解释**（实验 2 + 架构分析）：答案 token 信息密度 **29.5×** → Context 信息高度集中 → Q2C 的重分配功能冗余（信息已在正确位置，且被 Model Encoder 的 21 层 Self-Attention 替代）→ C2Q 的注入功能不可替代（Question 信号的唯一来源）→ 不对称性是 CQ Attention 运算结构的固有属性，跨架构稳健。放大效应源于 QANet 更强的后续编码器进一步压缩了 Q2C 的残余价值。

### 局限性

1. **子组件间交互**：A 和 C⊙A 存在代数依赖（C⊙A = C × A），单独零化的影响可能不反映独立贡献。方向级消融（同时零化 A + C⊙A）缓解了这一问题，且这正是与 BiDAF 对比的正确粒度。

2. **功能重叠度未直接测量**：我们论证 Q2C 的 Context 重分配功能被 Model Encoder 的 Self-Attention 替代，这一推理基于两者的运算定义（均为 Context 内部的加权聚合）和 Q2C 在 QANet 中近乎为零的因果效应（−1.03 F1）。但我们未直接测量两者的功能重叠度（如在移除 Model Encoder Self-Attention 后 Q2C 是否变得更重要）。这一交叉消融实验可进一步验证"功能替代"假说，但实现复杂度较高。

3. **S1 和 S2 共享参数**：两个方向的注意力矩阵来自同一个相似度矩阵 S 的不同归一化方向，共享可训练权重 w。两者不是真正独立的模块，而是同一计算的两个视角。
