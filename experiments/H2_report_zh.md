# H2：CQ Attention 的方向不对称性——从 BiDAF 到 QANet 的跨架构验证

## 动机

QANet 从 BiDAF（Seo et al., 2017）继承了双向 CQ Attention 设计。该层将 Context 和 Question 两条独立编码的信息流融合为一个表征，输出是四个子组件的拼接：

```
输出 = [C;  A;  C⊙A;  C⊙B]     ——共 4d 维
        ↑    ↑    ↑      ↑
       原始  C→Q  门控   Q→C→C
```

其中 A = S1·Q（C→Q 方向），C⊙B 来自 B = S1·S2ᵀ·C（Q→C 方向），S1 沿 Q 维度 softmax，S2 沿 C 维度 softmax。两个方向共享同一个相似度矩阵 S 和可训练权重 w，区别仅在于 softmax 的归一化方向。

BiDAF（Seo et al., 2017, Table 1b）通过训练时消融（移除组件后重训练模型）测量了两个方向的相对重要性：

| 条件 | F1 | ΔF1 |
|---|---|---|
| BiDAF (single) | 77.3 | — |
| 移除 C2Q (A) | 67.7 | −9.6 |
| 移除 Q2C (B) | 73.7 | −3.6 |

C2Q 方向的重要性约为 Q2C 的 **2.7 倍**。然而 BiDAF 使用 LSTM 作为编码器，QANet 用卷积和自注意力完全替换了 LSTM。**架构改变了，不对称性是否仍然成立？**

这一问题并非显然：QANet 的 Model Encoder（7 blocks × 3 passes = 21 层）比 BiDAF 的双层 LSTM 有远强的后续处理能力，可能能够从较弱方向的粗略信号中补偿信息。如果 QANet 的强编码器"抹平"了方向差异，则不对称性是 LSTM 容量瓶颈的产物；如果不对称性持续甚至加剧，则它是注意力融合机制的固有属性。

此外，BiDAF 的消融粒度为方向级（C2Q vs Q2C），未分离 [C, A, C⊙A, C⊙B] 四个子组件的独立贡献。我们有机会以更细的粒度重新审视这一问题。

## 研究问题

BiDAF 发现的 C2Q > Q2C 方向不对称性（2.7×）在 QANet 的 Conv+Attention 架构下是否成立？如果是，什么机制在不同架构下维持了它？

## 方法学说明：评估时干预与机制分析

BiDAF 的消融实验（Seo et al., 2017, Table 1b）采用**训练时消融**——移除组件后重新训练整个模型。这回答的是架构设计问题：*该组件在架构上是否可替代？*

我们选择 **eval 时消融**——在完整训练好的模型上直接零化子组件输出。这是一个有意识的方法论选择，而非资源限制下的退而求其次。原因如下：

我们的研究目标是**机制分析**——理解已训练模型中各子组件当前承担的功能角色——而非架构搜索。这与 Meng et al.（2022）在 GPT 中定位事实关联所采用的因果追踪方法论一致：在训练好的模型上施加干预（腐蚀 + 恢复），观察模型行为的变化，从而定位特定计算步骤的功能角色。训练时消融会让模型在训练过程中学会补偿，抹去被移除组件原本承担的功能痕迹，反而**不适合**机制分析。

因此，eval 时消融测量的是"模型当前如何使用该组件"——这正是我们需要回答的问题。

**与 BiDAF 的比较如何成立**：由于方法不同，两者的 ΔF1 **绝对值不可直接比较**（eval 时消融因分布偏移倾向于产生更大的性能下降）。但方法论偏差对 C2Q 和 Q2C 两个方向的影响近似等比例——两者都是注意力子组件，零化造成的分布偏移程度相当。因此 **C2Q/Q2C 方向比值**是跨方法稳健的比较指标。我们的比较严格限定在比值层面。

## 实验 2：什么机制维持了不对称性？

**前提**：实验 1 确认方向不对称性成立（如果不成立，本实验的问题变为"什么导致了不对称性的消失"，但实验设计不变）。

**问题**：C2Q 方向更重要。但 C2Q 和 Q2C 共享同一个相似度矩阵 S，区别仅在 softmax 方向。为什么仅仅改变归一化方向就产生了如此大的因果差异？一个可能的解释是：Context 中与答案预测相关的信息在空间上高度集中（答案区间仅占全部 token 的一小部分），C2Q 的 S1 矩阵（沿 ~30 个 Question token 做 softmax，归一化维度短）天然更聚焦，更善于精确指向这些高密度位置。

我们通过选择性腐蚀检验这一解释。

只对 Context 的特定位置施加噪声（σ = 3 × std），其余位置保持干净。200 样本，每样本 3 次噪声重复。

- **non\_ans\_only**：只腐蚀非答案、非填充位置

### 推理

- **如果密度比 >> 1**（例如 >5×）：Context 的因果信息高度集中在答案区间。C2Q 的结构性优势（S1 沿短维度 softmax → 更聚焦）使其更善于精确指向这些高密度位置，而 Q2C 的 S2（沿 ~400 个 Context token softmax → 更弥散）在定位精度上天然处于劣势。这一结构性差异独立于模型参数——它来自序列长度的不对称性，因此跨架构稳健。
- **如果密度比较低**（<2×）：Context 信息是分布式的。C2Q 的优势不在于空间定位精度，而可能在于查询对齐的信号质量。不对称性的机制需要其他解释。

### 局限性

- 腐蚀答案区间 token 直接移除了目标信号，高 TE 是预期的。真正有信息量的观测是非答案 token 的效应和密度比的具体数值。
- 选择性腐蚀改变的是输入端，不能直接证明"C2Q 子组件因此受到更大影响"。从信息集中度到方向优势的连接依赖一个结构性论证（S1 的归一化维度更短 → 更聚焦），而非直接的因果干预。我们在推理中明确标注了这一逻辑跳跃。

## 综合

### 回答

**研究问题**：BiDAF 的 C2Q > Q2C 方向不对称性在 QANet 下是否成立？什么机制维持了它？

**回答**：[根据实际数据填写]

**如果 C2Q >> Q2C（与 BiDAF 方向一致）**：

方向不对称性跨架构稳健。在 QANet 的 Conv+Attention 架构下，C2Q 子组件（A, C⊙A）的联合因果效应是 Q2C（C⊙B）的 [TBD] 倍（BiDAF 为 2.7×）。cq\_resizer 权重范数独立验证了这一排序。Context 的因果信息高度集中在答案区间（信息密度比 [TBD]×），C2Q 方向的结构性优势——S1 沿较短的 Question 维度归一化，天然更聚焦——使其更善于精确指向这些高密度位置。这一机制来自序列长度的不对称性，不依赖特定的编码器架构（LSTM 或 Conv+Attention），因此解释了不对称性的跨架构稳健性。

**如果 C2Q ≈ Q2C（与 BiDAF 不一致）**：

方向不对称性不跨架构稳健。QANet 的 Model Encoder（21 层深）比 BiDAF 的双层 LSTM 有远强的后续处理能力，能够从 Q2C 方向的弥散信号中提取充分信息，抹平了方向差异。不对称性是 LSTM 容量瓶颈的产物——当下游编码器足够强时，两个方向提供的信息可以被同等利用。

### 证据链

1. **前置验证**：CQ Attention 整体消融导致 F1 下降 [TBD]%，确认其信息瓶颈角色，解剖内部子组件有意义。
2. **核心事实**（实验 1）：子组件消融 + 方向级汇总确立 C2Q/Q2C 比值 = [TBD]×；cq\_resizer 权重从参数侧独立验证。
3. **机制解释**（实验 2）：答案 token 信息密度是非答案 token 的 [TBD] 倍 → Context 信息高度集中 → C2Q 的 S1（短维度 softmax）更善于聚焦高密度位置 → 解释不对称性的跨架构稳健性。

### 局限性

1. **子组件间交互**：A 和 C⊙A 存在代数依赖（C⊙A = C × A），单独零化的影响可能不反映独立贡献。方向级消融（同时零化 A + C⊙A）缓解了这一问题，且这正是与 BiDAF 对比的正确粒度。

2. **从信息集中度到方向优势的逻辑跳跃**：实验 2 证明的是"信息集中在答案区间"，连接到"C2Q 因此更有优势"依赖结构性论证（S1 归一化维度更短），而非直接因果干预。这一限制已在推理中明确标注。

3. **S1 和 S2 共享参数**：两个方向的注意力矩阵来自同一个相似度矩阵 S 的不同归一化方向，共享可训练权重 w。两者不是真正独立的模块，而是同一计算的两个视角。
