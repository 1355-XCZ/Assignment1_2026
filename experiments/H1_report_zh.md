# H1：QANet Model Encoder 组件级重要性分析

## 假设

> QANet Model Encoder 中，第二层卷积（Conv_1）是对答案预测因果上最重要的组件。这种重要性源于其结构化的信息内容——具体而言是通用局部特征提取——而非答案位置编码或输出幅度。

## 背景

QANet 的 Model Encoder 由 7 个堆叠的 Encoder Block 组成，重复 3 遍（产生 M1、M2、M3）。每个 block 包含 4 个子层：

```
Position Encoding → Conv_0 → Conv_1 → Self-Attention → FFN
```

每个子层有残差连接：`output = residual + sublayer(x)`。

Conv_0 和 Conv_1 架构完全相同（depthwise separable convolution，相同 kernel size 和维度）。QANet 论文 Table 5 报告移除所有卷积层后重训练导致 −2.7 F1，移除 Self-Attention 导致 −1.3 F1。但论文未区分 Conv_0 和 Conv_1，也未探究差异背后的机制。

---

## 实验 1：全局消融（Eval-time 组件移除）

**方法**：将某类组件在所有 7 blocks × 3 passes 中的输出零化。在完整 dev set（10,465 样本）上评估 F1/EM。

**机制**：子层输出在残差加法前被替换为零，因此下游层只收到残差（skip connection）。

| 配置 | F1 | ΔF1 | EM | 移除层数 |
|---|---|---|---|---|
| baseline | 70.24 | — | 59.22 | 0 |
| skip_conv_1 | 37.70 | −32.54 | 23.59 | 21 |
| skip_attn | 65.44 | −4.81 | 55.18 | 21 |
| skip_ffn | 66.92 | −3.32 | 54.56 | 21 |
| skip_conv_0 | 69.16 | −1.09 | 57.85 | 21 |
| skip_conv（两者） | 27.14 | −43.11 | 13.60 | 42 |

**发现 1**：重要性排序为 **Conv_1（−32.54）>> Attn（−4.81）> FFN（−3.32）> Conv_0（−1.09）**。Conv_1 移除是灾难性的；Conv_0 移除几乎无影响。

**发现 2**：Conv_0 + Conv_1 同时移除（−43.11）超过各自移除之和（−32.54 + −1.09 = −33.63），呈超可加性交互（+9.48）。当 Conv_0 也缺失时，Conv_1 的缺失造成更大破坏。

**局限**：零化消融同时移除了组件的信息和 norm 贡献，无法区分哪个因素驱动了损害。

---

## 实验 2：ROME 风格因果追踪

**方法**：改编自 Meng et al.（2022）。每个样本三步：
1. **Clean run**：收集所有子层激活，记录 P(正确 span)。
2. **Corrupted run**：向 context 嵌入添加高斯噪声（σ = 3× 输入标准差），记录退化后的 P(正确 span)。
3. **Corrupted-with-restoration**：重新运行 corrupted 输入，但恢复某一个子层的 clean 激活，测量恢复量（Average Indirect Effect, AIE）。

**配置**：200 样本（20 预跑），3 次噪声重复，84 个子层被追踪（7 blocks × 3 passes × 4 组件）。

### 结果

| 组件 | Mean AIE | Sum AIE | 占总量比例 |
|---|---|---|---|
| Conv_1 | 0.0210 | 0.4418 | 67.1% |
| Conv_0 | 0.0047 | 0.0986 | 15.0% |
| FFN | 0.0040 | 0.0844 | 12.8% |
| Self-Attn | 0.0016 | 0.0343 | 5.2% |

按 pass 聚合：Pass 0（M1）= 0.3395 > Pass 1（M2）= 0.2324 > Pass 2（M3）= 0.0871。

按 block 深度聚合：Block 0（0.2282）> Block 1（0.1574）> ... > Block 6（0.0371），单调递减。

**发现 3**：因果追踪确认 Conv_1 主导地位。恢复 Conv_1 的 clean 激活恢复了最多信息。效应集中在浅层 block 和早期 pass。

---

## 实验 3：Norm 诊断

**动机**：子层输出 L2 norm 与 AIE 的相关性可能表明因果追踪在测量幅度而非重要性。

**方法**：在 200 次 clean forward pass 中计算每个子层输出的 L2 norm。

| 组件 | 平均 Norm | 相对 Conv_0 比率 |
|---|---|---|
| Conv_1 | 250.47 | 2.22x |
| Conv_0 | 112.77 | 1.00x |
| FFN | 103.50 | 0.92x |
| Self-Attn | 81.84 | 0.73x |

**84 个子层的 Pearson r(norm, AIE) = 0.895。** 80% 的 AIE 方差可由输出 norm 解释。

### 这是混淆因素吗？

**对于 Conv_1 vs Attn**：两者的 norm 和 AIE 都偏向 Conv_1，无法分离。

**对于 Attn vs FFN vs Conv_0**：norm 排序为 Conv_0（113）> FFN（104）> Attn（82），但消融损害排序为 Attn（−4.81）> FFN（−3.32）> Conv_0（−1.09）。**Norm 和重要性反向相关。** 这是这三个组件排序不受 norm 驱动的硬证据。

**发现 4**：Attn > FFN > Conv_0 的排序对 norm 效应是稳健的（反相关）。Conv_1 优于 Attn 的幅度因共享混淆因素而不确定。

---

## 实验 4：零化 vs 噪声替换

**动机**：直接分离信息和幅度。

**方法**：对比两种干预：
- **零化**：将输出替换为零（移除信息和 norm）。
- **噪声替换**：将输出替换为相同 L2 norm 的随机噪声（移除信息，保留 norm）。

**逻辑**：
- 若 norm 驱动重要性 → 噪声（保留 norm）应比零化损害更小。
- 若信息驱动重要性 → 噪声应与零化损害相当或更大（噪声也移除了信息，且额外引入干扰）。

| 组件 | 零化 ΔF1 | 噪声 ΔF1 | 噪声/零化比率 |
|---|---|---|---|
| Conv_1 | −29.27 | −41.29 | 1.41x |
| Conv_0 | −2.23 | −2.70 | 1.21x |
| Attn | −5.20 | −5.89 | 1.13x |
| FFN | −3.90 | −4.72 | 1.21x |

（在 2,048 子采样 dev set 上评估。）

**发现 5**：噪声替换对所有组件的损害都大于零化。这否定了 norm 假说。如果 norm 是关键，保留它（通过噪声）应该有帮助。实际上，norm 匹配的噪声主动破坏了残差流，因为它注入了干扰下游处理的结构性垃圾。

**解释**：四个组件都产生结构化的有用信息。移除造成的损害来自信息的丢失，而非幅度的丢失。Conv_1 的噪声/零化比率最高（1.41x），因为其大 norm 意味着注入的垃圾信号影响最大。

---

## 实验 5：线性探针——Conv_1 编码了什么信息？

**动机**：既然 Conv_1 的信息是关键的（实验 4），这些信息是什么？具体来说，Conv_1 是否编码了答案位置信号？

**方法**：在每个子层的 per-token 输出向量上训练逻辑回归分类器。标签：1 = token 在答案 span 内，0 = 其他。使用 ROC-AUC 评估（对类别不平衡稳健，正类比例约 2.3%）。500 个样本，80/20 训练/测试划分，在 5 个代表性（pass, block）位置进行探测。

### 结果：按组件类型平均 AUC

| 组件 | Mean AUC（5 个 block） |
|---|---|
| Conv_1 | 0.901 |
| FFN | 0.898 |
| Self-Attn | 0.894 |
| Conv_0 | 0.876 |

### 结果：按 block 位置平均 AUC（4 个组件平均）

| 位置 | Mean AUC |
|---|---|
| p0_b0 | 0.822 |
| p0_b3 | 0.901 |
| p0_b6 | 0.907 |
| p1_b0 | 0.910 |
| p2_b0 | 0.922 |

**发现 6**：四个组件对答案位置信息的编码能力相近。Conv_1 的 AUC 优势仅 +0.007（相对于 Attn）。这与消融结果形成鲜明对比——消融中 Conv_1 的移除损害是 Attn 的 7 倍。

**发现 7**：答案位置信息随深度积累（AUC 从 p0_b0 的 0.822 增至 p2_b0 的 0.922），且不集中于任何单一组件。

**解释**：Conv_1 的重要性并非来自编码"答案在哪里"。所有子层对这一信息的编码能力相当。Conv_1 的关键角色必然在别处——在于其通用局部特征的质量和结构。

---

## 实验 6：逐 Block 消融

**方法**：在单个（pass, block）中移除 conv（conv_0 和 conv_1 一起）或 self_attn，在 2,048 子采样 dev set 上测量 F1。共 42 个配置。子集 baseline F1 = 68.01。

### 关键结果

| 配置 | F1 | ΔF1 |
|---|---|---|
| p0_b0_conv | 66.73 | −1.28 |
| p0_b0_self_attn | 67.35 | −0.66 |
| 其他所有配置 | ~67.7–68.2 | −0.3 至 +0.2 |

**发现 8**：单个 block 移除影响极小（最大 ΔF1 = −1.28）。仅 p0_b0_conv 显示明确显著的效果。大多数 block 是单独可替代的。

**发现 9**：集体移除是灾难性的（所有 conv 移除 −43.11），但单个移除可忽略不计。这揭示了**分布式冗余架构**：每个 block 贡献微小增量，单独可有可无但集体不可或缺。7 block × 3 pass 的设计提供了大量冗余。

**发现 10**：按 pass 平均 ΔF1：M1（−0.23）>> M2（−0.03）>> M3（+0.01）。Pass 2（M3）的 block 可以被单独移除而无损甚至微升，表明在单 block 级别上其边际计算价值为零。

---

## 综合：Conv_1 的角色是什么？

综合六个实验的证据：

| 证据 | 说明了什么 |
|---|---|
| 全局消融 | Conv_1 移除是灾难性的（−32.54 F1） |
| 因果追踪 | Conv_1 携带最多可恢复信息（67% AIE） |
| Norm 诊断 | Conv_1 输出 norm 是 Conv_0 的 2.2 倍；r(norm, AIE)=0.895 |
| 噪声替换 | 重要的是信息而非 norm（噪声损害 > 零化损害） |
| 线性探针 | Conv_1 并不特别编码答案位置 |
| 逐 block 消融 | 单个 block 可替代；集体效应才是关键 |

**结论**：Conv_1 在 QANet Model Encoder 中充当**主要的局部特征放大器**。它产生高 norm、信息丰富的表示，作为下游 Self-Attention 和 FFN 处理的基底。它的关键作用不在于检测答案边界（所有组件对此编码能力相当），而在于提供基础性的局部特征——没有这些特征，整个下游处理链路退化。

Conv_0 尽管架构相同，但学到的是最小化的预处理角色（低 norm、低消融影响）。Conv_0 和 Conv_1 之间的不对称性来自训练动态：模型收敛到一种配置，其中一层卷积承担主要工作，另一层接近冗余。

### 局限性

1. **Conv_1 vs Attn 的幅度**：Conv_1 相对于 Attn 的精确优势倍数不确定（2–7x 范围），因为两者共享 norm 相关性混淆。
2. **Eval-time 消融 vs 重训练**：我们的零化干预比论文的"移除后重训练"方法更激进，绝对损害数值被放大。
3. **探针范围**：线性探针仅测试答案位置编码。Conv_1 可能编码其他类型信息（句法模式、实体边界、局部上下文），二元答案/非答案探针无法检测到这些。
4. **因果追踪样本量**：20 个预跑样本的置信区间较宽。完整的 200 样本运行会加强估计。

### 原创贡献

1. **Conv_0 vs Conv_1 分离**：首次证明两个架构相同的堆叠卷积学到了截然不同的角色（移除分别导致 −1.09 vs −32.54 F1）。
2. **噪声替换控制实验**：确立组件重要性反映信息内容而非输出幅度，解决了 norm 混淆问题。
3. **线性探针否定结果**：表明 Conv_1 的重要性不在于答案位置编码，排除了最直观的解释，指向通用特征提取。
4. **分布式冗余量化**：量化了单 block 可替代性（最大 −1.28 F1）与集体必要性（−43.11 F1）之间的差距，刻画了 QANet 的容错架构。
